# TRACE Roadmap

This roadmap reflects the implementation direction required to make TRACE desirable to digital forensic companies and defensible in real casework.

## Near-term

- add stronger examiner override rationale capture
- add richer manifest provenance and package integrity checks
- add response caching inspection and replay tooling
- add better parser support for court transcript formats

## Mid-term

- add Cellebrite UFED XML parser
- add Magnet AXIOM JSON parser
- add rendered PDF report output
- add larger gold-standard validation corpus
- add performance and throughput profiling for long transcripts

## Longer-term

- case batch processing
- signed evidence-package manifests
- team workflow support for dual-coder review
- configurable deployment modes for labs with strict network controls

## Success standard

TRACE should become:

- faster than spreadsheet-based manual coding
- more reproducible than ad hoc workflows
- easier to defend than informal LLM-assisted analysis
- easier to integrate into current forensic lab process than custom internal tooling
